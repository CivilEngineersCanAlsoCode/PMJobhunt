# /edit-template — Base Template Edit Workflow

> ⚠️ `Templates/Base_Template.html` is **IMMUTABLE** during normal resume customization.
> This workflow is the **only authorized path** to modify it.

---

## When to Use

Only trigger this workflow when you want to permanently change the layout, CSS variables, column widths, or structure of the base HTML resume template. Changes here will affect ALL future resumes generated by Sync.

---

// turbo-all

## Step 1 — Show Current Template Version

Run:

```bash
cd /path/to/PM
git log --oneline Templates/Base_Template.html | head -5
```

Print the last 5 commits that touched the base template so the user can see the version history before making changes.

## Step 2 — Require Explicit User Confirmation

Print the following and wait for user to explicitly type `YES I WANT TO EDIT THE TEMPLATE`:

> "⚠️ WARNING: You are about to edit `Templates/Base_Template.html`.
>
> This file is the foundation for ALL Sync resumes. Any layout or CSS change here will affect every future resume generated by the system.
>
> **Existing customized resumes will NOT be retroactively affected** — they are already committed to GitHub.
>
> To proceed, type exactly: `YES I WANT TO EDIT THE TEMPLATE`
> To cancel, type anything else."

If the user does not type the exact confirmation phrase, abort immediately and print: "Template edit cancelled. No changes made."

## Step 3 — Create Versioned Backup

Before any changes, create a dated backup:

```bash
cp Templates/Base_Template.html Templates/Base_Template_backup_YYYY-MM-DD.html
git add Templates/Base_Template_backup_YYYY-MM-DD.html
git commit -m "backup: Base_Template before edit session $(date +%Y-%m-%d)"
```

## Step 4 — Describe the Intended Change

Ask the user: "What do you want to change? Describe the edit in one sentence."

Record the description — it will be used as the git commit message.

## Step 5 — Apply the Edit

Make the requested changes to `Templates/Base_Template.html`.

**Mandatory CSS guardrails that must NEVER be removed or violated:**

| Rule               | Value                                                   |
| ------------------ | ------------------------------------------------------- |
| Date column width  | `.c5 ≥ 14.5%`                                           |
| Date cell wrapping | `white-space: nowrap` on `.yr` and `.sub td:last-child` |
| Table border color | `--color-border: #000000`                               |
| Body font          | `'Segoe UI', Calibri, Arial, sans-serif` (Sans-Serif)   |
| Base font size     | `9pt` minimum                                           |
| Sub-header style   | Pipe approach — no heavy dark backgrounds               |
| Bullet indent      | `padding-left: 14px` maximum                            |

If the requested change would violate any guardrail above, abort and print: "This change would violate a core layout guardrail. Edit cancelled. Please rephrase your request."

## Step 6 — Visual Verification

Print:

> "Please open `Templates/Base_Template.html` in your browser and verify the layout looks correct. Type `LOOKS GOOD` to commit, or describe what's wrong to make further adjustments."

Wait for user confirmation before committing.

## Step 7 — Commit the Change

```bash
git add Templates/Base_Template.html
git commit -m "template: <user-described change from Step 4>"
git push
```

## Step 8 — Update Guardrail Log

Append a line to `Templates/TEMPLATE_CHANGELOG.md`:

```
| YYYY-MM-DD | <user-described change> | Backup: Base_Template_backup_YYYY-MM-DD.html |
```

Push the changelog:

```bash
git add Templates/TEMPLATE_CHANGELOG.md
git commit -m "chore: update template changelog"
git push
```

## Step 9 — Done

Print:

> "✅ Base template updated and committed.
> Change: <description>
> Backup saved at: `Templates/Base_Template_backup_YYYY-MM-DD.html`
> All future Sync resumes will use the updated template.
> Existing resumes in `Sync/` are unaffected."
